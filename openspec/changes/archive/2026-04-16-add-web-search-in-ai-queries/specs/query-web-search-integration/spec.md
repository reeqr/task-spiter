## ADDED Requirements

### Requirement: 全局联网搜索开关
系统必须提供一个全局联网搜索开关，并且在开关关闭时保持现有 AI 请求行为。

#### Scenario: 全局开关关闭时不注入联网工具
- **WHEN** 用户未启用联网搜索
- **THEN** 系统必须按现有 Completion 请求结构发送消息，不包含 `tools.web_search` 参数

#### Scenario: 全局开关开启时所有请求尝试联网
- **WHEN** 用户启用联网搜索
- **THEN** 系统必须对所有 AI 请求优先尝试联网增强路径

### Requirement: 全局联网参数配置
系统必须支持配置全局联网搜索的基础参数，并在请求中按配置生效。

#### Scenario: 配置检索条数
- **WHEN** 用户设置联网搜索结果条数
- **THEN** 系统必须在联网请求中使用该条数参数并约束为有效范围

#### Scenario: 配置来源过滤
- **WHEN** 用户设置域名过滤条件
- **THEN** 系统必须在联网请求中带上域名过滤参数

#### Scenario: 配置时间范围过滤
- **WHEN** 用户设置时间范围过滤条件
- **THEN** 系统必须在联网请求中带上对应时间过滤参数

### Requirement: 不支持或失败时自动降级
系统必须在联网能力不可用时自动降级为普通请求，且不得中断用户查询流程。

#### Scenario: 静态能力表判定不支持
- **WHEN** 当前 provider 在能力表中被标记为不支持 `web_search`
- **THEN** 系统必须跳过联网工具注入并发送普通请求

#### Scenario: 运行时探测判定不支持
- **WHEN** 联网请求返回“工具不支持”或等效错误
- **THEN** 系统必须将该 provider 标记为运行时不支持并回退到普通请求路径

#### Scenario: 联网调用失败
- **WHEN** 联网请求因参数或远端错误失败
- **THEN** 系统必须自动回退到普通请求路径，并向用户给出可理解提示

### Requirement: 联网回答来源可追溯
系统必须在联网增强回答中提供可追溯来源信息，至少包含链接与时间信息。

#### Scenario: 联网回答包含来源信息
- **WHEN** 系统使用联网搜索增强生成回答
- **THEN** 回答结果或关联展示区域必须提供来源链接与对应时间信息

#### Scenario: 无可用来源时提示
- **WHEN** 联网模式下未获取到有效来源信息
- **THEN** 系统必须明确提示“未获取到可追溯来源”，不得伪造来源

### Requirement: 覆盖所有 AI 请求类型
系统必须将全局联网开关应用到所有 AI 请求类型，而不仅限于术语问答。

#### Scenario: 概念拆解请求应用全局开关
- **WHEN** 用户触发概念拆解请求且全局联网开关开启
- **THEN** 系统必须优先尝试联网增强路径并在失败时自动降级

#### Scenario: 任务拆解请求应用全局开关
- **WHEN** 用户触发任务拆解请求且全局联网开关开启
- **THEN** 系统必须优先尝试联网增强路径并在失败时自动降级
