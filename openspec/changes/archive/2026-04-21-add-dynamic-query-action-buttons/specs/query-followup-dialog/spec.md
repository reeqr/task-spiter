## MODIFIED Requirements

### Requirement: 查询结果弹窗多轮追问
系统 MUST 在所有查询动作结果弹窗中提供可连续追问的对话能力。

#### Scenario: 首轮结果进入会话流
- **WHEN** 用户触发任一查询动作并收到首轮 AI 结果
- **THEN** 系统 MUST 打开对应弹窗并将首轮结果作为当前会话中的第一条 AI 消息展示

#### Scenario: 连续追问展示
- **WHEN** 用户在弹窗中提交新的追问
- **THEN** 系统 MUST 将该问题与后续 AI 回复按时间顺序追加到同一会话流

### Requirement: 追问发送规则
系统 MUST 采用 Enter 作为追问发送快捷方式，并在请求进行中避免重复发送。

#### Scenario: Enter 发送
- **WHEN** 用户在追问输入框按下 Enter 且输入有效
- **THEN** 系统 MUST 触发一次追问请求并清空输入框

#### Scenario: 发送中防重入
- **WHEN** 当前追问请求尚未完成
- **THEN** 系统 MUST 禁止再次触发发送并提供进行中状态反馈

### Requirement: 会话关闭即清空
系统 MUST 在关闭查询结果弹窗时清空该弹窗会话状态。

#### Scenario: 关闭后清空历史
- **WHEN** 用户关闭任一查询结果弹窗
- **THEN** 系统 MUST 清空该弹窗会话消息、输入内容与发送态

#### Scenario: 重新打开为新会话
- **WHEN** 用户再次打开同一查询动作结果弹窗
- **THEN** 系统 MUST 以新的首轮结果重新开始会话，不得复用已关闭会话的历史

### Requirement: 不同查询动作会话隔离
系统 MUST 保证不同查询动作的会话上下文互相独立。

#### Scenario: 动作级上下文隔离
- **WHEN** 用户针对同一术语在不同查询动作弹窗中分别进行追问
- **THEN** 系统 MUST 各自维护独立消息历史，不得共享或污染上下文
