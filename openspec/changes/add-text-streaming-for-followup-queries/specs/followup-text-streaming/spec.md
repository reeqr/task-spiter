## ADDED Requirements

### Requirement: 追问文本流式输出
系统 MUST 在“解释追问”和“出题角度追问”场景中支持文本流式输出，用户可以在响应生成过程中持续看到新增内容。

#### Scenario: 追问后增量显示
- **WHEN** 用户在任一查询弹窗提交追问
- **THEN** 系统 MUST 以文本增量方式持续更新同一条助手消息，而不是等待完整结果后一次性展示

#### Scenario: 流结束后收敛状态
- **WHEN** 流式响应结束
- **THEN** 系统 MUST 将当前追问状态标记为完成，并保留完整最终文本

### Requirement: 流式回退兼容
系统 MUST 在模型或 provider 不支持流式输出时自动回退到原非流式请求，且保持追问可用。

#### Scenario: 不支持流式自动回退
- **WHEN** 当前请求返回流式不支持或等效错误
- **THEN** 系统 MUST 自动改为非流式请求并返回完整回答

#### Scenario: 流式中断回退提示
- **WHEN** 流式响应在中途异常中断
- **THEN** 系统 MUST 保留已生成文本并向用户提示本次响应已中断
